# Skin Trading Platform

Учебный проект для торговли игровыми скинами.  
Состоит из двух Spring Boot сервисов:
- `marketplace` — основной API маркетплейса (пользователи, скины, лоты, покупка).
- `notification` — отправка уведомлений о событиях покупки (уведомление не отправляется, а просто логируется отправка).

## Запуск проекта

```bash
docker compose up -d --build
```

## Swagger marketplace сервиса

```
http://localhost:8080/swagger-ui/index.html
```

## Что умеет проект

- Регистрация и логин пользователей с JWT.
- Ролевая модель (`ADMIN` / `USER`) и разграничение доступа к endpoint-ам.
- CRUD по скинам (создание/изменение/удаление доступно администратору).
- Создание лотов на маркетплейсе.
- Покупка лота с проверками бизнес-ограничений:
  - нельзя купить свой лот;
  - нельзя купить неактивный лот;
  - нельзя купить при недостатке баланса.
- Пополнение баланса пользователя.
- Пагинация для списков скинов и лотов.
- Асинхронная публикация событий покупки в Kafka и обработка в сервисе уведомлений.

## Архитектура и устройство

### Модули

- `marketplace`
  - REST API для CRUD со скинами и созданием/покупкой скинов(лоты).
  - Работает с БД `market_db`.
  - Пишет события в таблицу outbox.
  - Периодически отправляет события в топик `purchases`.
- `notification`
  - Подписывается на топик `purchases`.
  - Реализует идемпотентную обработку событий.
  - Работает с БД `notification_db`.

## Также используется

- **Transactional outbox pattern**  
  При покупке лота событие сначала пишется в `outbox_events` в рамках той же транзакции, что и изменение бизнес-данных. Далее отдельный планировщик отправляет событие в Kafka.  
  Если отправка в Kafka не удалась несколько раз подряд (константа в планировщике), событие помечается как dead letter и больше не обрабатывается планировщиком.

- **Idempotent consumer**  
  `notification` сервис хранит обработанные сообщения в `processed_messages` (уникальны по `aggregate_id + event_type`) и не обрабатывает событие повторно.

- **Optimistic lock**  
  Используется для изменения баланса пользователя и обновления скинов.

- **Pessimistic lock**  
  Используется при покупке лота, чтобы избежать race condition.
